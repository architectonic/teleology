# Start Here

This is the first-run guide for agents reading `teleology`.

## First Run

1. Read `AGENTS.md` for repository rules.
2. Read `README.md` for package purpose and boundaries.
3. Read `index.md` for progressive disclosure.
4. Read `doctrine/index.md` for available doctrine concepts.
5. Read `doctrine/source-hierarchy.md` before making factual claims.
6. Read `doctrine/epistemology.md` before reasoning from memory, summaries, or inference.
7. Read `doctrine/operating-loop.md` before making changes.

## Chain of Truth

```text
Memory or prior chat = hints
Knowledge bundle = curated index and operating context
Source artifacts = truth
```

Never skip the chain. Memory tells you where to look. The bundle tells you what has been curated. Source artifacts tell you what is true.

## When To Ask Before Acting

Ask before acting when:

- the target repository or file is ambiguous;
- the change would expose private context;
- the change touches credentials, secrets, auth, billing, production, or user data;
- the request requires inventing missing requirements;
- the requested update conflicts with this repository's doctrine.

Proceed without extra confirmation when the target is explicit, the change is reversible, the source basis is clear, and the update is small.

## Canonical Loop

```text
Read → Classify → Inspect → Plan → Act → Verify → Reconcile → Handoff
```

Use this loop for all durable changes.
